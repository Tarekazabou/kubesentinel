Six integrated modules that cover the full security lifecycle—from pre-deployment checks to post-incident forensics.

    


      

        

          
        

        
Static Policy Engine

        
Scans Kubernetes YAML, Helm charts, and Dockerfiles for misconfigurations before they reach your cluster. Catches privilege escalation, missing resource limits, root containers, and more.

        
YAMLHelmDockerfile

        
      


      

        

          
        

        
Runtime Monitor

        
Ingests and processes Falco security events in real time. Streams from a Unix socket or stdin pipeline and enriches each event with namespace, workload, and severity context.

        
FalcoReal-timeKubernetes

        
      


      

        

          
        

        
AI Behavioral Analyzer

        
Python ML service using Isolation Forest to detect anomalous workload behavior. Scores events and surfaces outliers without relying on static signatures.

        
PythonIsolation ForestAnomaly

        
      


      

        

          
        

        
Forensic Vault

        
Stores structured incident evidence with configurable retention, max-size pruning, and optional gzip compression. Keeps your audit trail lean and ready for investigation.

        
gzipRetentionEvidence

        
      


      

        

          
        

        
Report Generator

        
Produces investigation outputs in Markdown, JSON, and HTML. Generates incident timelines, violation summaries, and remediation notes from stored forensic evidence.

        
MarkdownJSONHTML

        
      


      

        

          
        

        
Gemini Enrichment

        
Optional Google Gemini integration that classifies runtime incidents and generates narrative report sections—with redaction safeguards and a deterministic fallback when the API is unavailable.

        
GeminiOptionalLLM

Static Policy Engine

        
Scan Kubernetes manifests against 50+ built-in security rules before deployment. Catches critical misconfigurations such as privileged containers, missing resource limits, and host namespace sharing.

        
    
          
  • Supports YAML, Helm, and Dockerfile
    • 
          
  • Configurable severity thresholds

    •

Command Reference

    
Every command you need, in one place.

    

      
      
      
      
    


    

      

        
Perform static analysis on Kubernetes manifests, Helm charts, and Dockerfiles. Detects misconfigurations before deployment.

      

      

        

          

            kubesentinel scan --path ./manifests
            
          

          Scan all YAML files in a directory
        

        

          

            kubesentinel scan --path ./deploy --severity high
            
          

          Filter results to HIGH and CRITICAL violations only
        

        

          

            kubesentinel scan --path ./deploy --rules ./my-rules.yaml
            
          

          Include custom rule definitions
        

        

          

            kubesentinel scan --path ./deploy --format json
            
          

          Output results as JSON (for CI pipelines)
        

      

    


    

      

        
Stream and process Falco security events in real time. Supports Unix socket and stdin pipeline modes.

      

      

        

          

            kubesentinel monitor
            
          

          Start monitoring with default config
        

        

          

            kubesentinel monitor --namespace production --deployment api
            
          

          Filter events to a specific workload
        

        

          

            kubesentinel monitor --source=socket
            
          

          Ingest Falco events via Unix socket
        

        

          

            kubectl logs -n falco -l app=falco -f | kubesentinel monitor-stdin
            
          

          Pipe kubectl logs directly into KubeSentinel
        

      

    


    

      

        
Generate investigation reports from forensic evidence. Supports Markdown, JSON, and HTML output formats.

      

      

        

          

            kubesentinel report --incident-id <id>
            
          

          Generate a report for a specific incident
        

        

          

            kubesentinel report --from "2026-03-01" --to "2026-03-31" --format markdown,json
            
          

          Date-range report in multiple formats
        

        

          

            kubesentinel report --incident-id <id> --format html --no-llm
            
          

          HTML report without Gemini enrichment
        

      

    


    

      

        
KubeSentinel is configured via a YAML file. Below are the key configuration sections.

      

      

        

          config.yaml
          
        

        
forensics:
  storage_path: "./forensics"
  retention_days: 90
  max_size_mb: 1000
  compression: true

reporting:
  formats: ["json", "markdown", "html"]
  output_path: "./reports"

gemini:
  enabled: false
  classify_runtime: false
  api_key: ""
  model: "gemini-2.5-flash"
  timeout_seconds: 15
